How It Works

Three lines of code to start saving

1

Send Your Prompt

Make a GET request with your prompt. We'll hash it and check the cache.

# Check cache (GET) curl "https://api.promptcache.dev/cache" \ -H "X-API-Key: your-api-key" \ -d "prompt=Your prompt here"
2

Cache Hit or Miss

If we've seen it before, instant response. If not, call your LLM and cache the result.

// Cache hit response { "cached": true, "response": "...", "hits": 42, "age": 3600000 }
3

Save Up to 90%

Repeated prompts are served from cache. Watch your API costs plummet.

# Cache a response (POST) curl -X POST "https://api.promptcache.dev/cache" \ -H "X-API-Key: your-api-key" \ -d '{"prompt":"...","response":"...","ttl":3600000}'

API Reference

Simple REST API, ready to integrate

Endpoints

GET /cache/:prompt

Retrieve cached response for a prompt. Returns cached: true if hit, cached: false if miss.

POST /cache

Store a prompt-response pair in the cache with optional TTL (default: 1 hour).

GET /stats

Get cache statistics including hit rate, total entries, and backend info.

DELETE /cache

Clear all cached entries or a specific entry by prompt.
